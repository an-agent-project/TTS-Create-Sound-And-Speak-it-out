# 启动说明

本文说明如何在本地启动前端、后端，以及初始化 MySQL 数据库。

## 1. 环境要求

- Node.js：用于启动 Vue + Vite 前端。
- Python 3.11+：用于 FastAPI 后端。
- MySQL 8.0：用于业务数据库。
- Windows PowerShell：以下命令按 PowerShell 编写。

## 2. 项目目录

进入项目根目录：

```powershell
cd C:\Users\lenovo\Desktop\TTS\TTS-Create-Sound-And-Speak-it-out
```

## 3. 前端启动

安装前端依赖：

```powershell
npm install
```

启动前端开发服务：

```powershell
npm run dev
```

默认访问地址：

```text
http://localhost:5173
```

前端的 Vite 配置已经把以下请求代理到后端：

```text
/api -> http://127.0.0.1:8000
/static -> http://127.0.0.1:8000
/audio -> http://127.0.0.1:8000
```

因此前端调用试听接口时，可以直接请求：

```text
/api/tts/preview
```

## 4. 后端虚拟环境

项目后端应使用项目根目录下的 `.venv`，不要直接使用 Anaconda base 环境。

创建虚拟环境：

```powershell
C:\Users\lenovo\anaconda3\python.exe -m venv .venv
```

安装后端依赖：

```powershell
.\.venv\Scripts\python.exe -m pip install -r backend\requirements.txt
```
配置数据库连接。当前本地 MySQL 可使用：

```powershell
$env:DATABASE_URL="mysql+pymysql://root:123456@127.0.0.1:3306/tts_podcast?charset=utf8mb4"
```

## 5. 后端启动

启动 FastAPI：

```powershell
.\.venv\Scripts\python.exe -m uvicorn app.main:app --app-dir backend --host 127.0.0.1 --port 8000
```

健康检查：

```text
http://127.0.0.1:8000/health
```

接口文档：

```text
http://127.0.0.1:8000/docs
```

当前已经实现的后端接口：

```text
# Auth
POST /api/auth/register    # 注册新账号，返回 JWT
POST /api/auth/login        # 登录，返回 JWT
GET  /api/auth/me           # 获取当前用户信息（需 Bearer token）
PUT  /api/auth/me           # 更新当前用户信息（手机/邮箱/头像）
POST /api/auth/change-password  # 修改密码（需 Bearer token）

# TTS
POST /api/tts/preview       # 音色试听（含数据库缓存）

# Voices
GET    /api/voices          # 音色列表（匿名仅见默认，登录后可见个人专属）
GET    /api/voices/{voice_id}  # 音色详情
POST   /api/voices          # 新增专属音色（需登录，自动归属当前用户）
PUT    /api/voices/{voice_id}  # 更新音色（仅 owner 可编辑）
DELETE /api/voices/{voice_id}  # 软删除音色（仅 owner 可删除）
```

## 用户认证机制

系统采用 JWT (JSON Web Token) 认证：

- 用户密码使用 bcrypt 哈希存储在 `users` 表，不存明文。
- 注册/登录成功后返回 `access_token`（有效期 24 小时）。
- 前端将 token 存储在 `localStorage` 的 `auth_token` 键中。
- 后续请求在 HTTP Header 中带 `Authorization: Bearer <token>`。
- 后端通过 `get_current_user` 依赖解析 token 并返回当前用户。

## 个人专属音色机制

`voices` 表通过 `owner_id` 字段区分系统默认音色与用户专属音色：

| `owner_id` | 含义 | 可见范围 | 编辑/删除 |
|-----------|------|---------|----------|
| `NULL` | 8 个系统默认音色 | 所有用户（含未登录） | ❌ 仅管理员 |
| 具体 user id | 用户个人专属音色 | 仅该用户自己 | ✅ 音色所有者 |

- 匿名用户 `GET /api/voices` 仅返回 8 个默认音色。
- 登录用户返回默认音色 + 自己创建的音色。
- 用户 A 看不到用户 B 的专属音色。
- `PUT` / `DELETE` 系统默认音色返回 403 Forbidden。

## 6. 数据库初始化

数据库采用 MySQL 8.0。初始化脚本位于：

```text
backend/db/init.sql
```

执行初始化：

```powershell
mysql -u root -p < backend\db\init.sql
```

执行后会创建数据库：

```text
tts_podcast
```


默认开发管理员账号：

```text
用户名：admin
密码：Admin@123456
邮箱：admin@example.local
```

该账号由 `init.sql` 直接写入 `users` 表，密码为 bcrypt 哈希存储。非本地环境首次登录后请立即修改密码。

并创建以下表：

```text
users                       # 用户账号表
voices                      # 业务音色表
voice_provider_profiles     # TTS 供应商映射表
voice_preview_audios        # 试听音频缓存表
```

同时会写入当前音色库中的 Edge-TTS 音色种子数据，例如：

```text
晓晓 -> zh-CN-XiaoxiaoNeural
云希 -> zh-CN-YunxiNeural
晓伊 -> zh-CN-XiaoyiNeural
```

## 百炼 Qwen-TTS 配置

项目默认在个人音色库中提供 8 个百炼 Qwen-TTS 系统音色：`芊悦`、`苏瑶`、`晨煦`、`千雪`、`月白`、`四月`、`凯`、`阿闻`。这些音色不依赖本地 Qwen 模型，不需要 GPU；试听和一键生成配音时由后端按 Qwen-TTS 官方路径调用 `dashscope.MultiModalConversation.call`。

本地启动前，在 PowerShell 中配置 API Key：

```powershell
$env:DASHSCOPE_API_KEY="你的百炼API_KEY"
```

如果要使用声音复刻/音色克隆，还需要配置百炼工作空间，或直接配置自定义任务地址：

```powershell
$env:DASHSCOPE_WORKSPACE="你的Workspace ID"
# 或：
$env:BAILIAN_TTS_CUSTOMIZATION_URL="https://<workspace>.<region>.maas.aliyuncs.com/api/v1/services/audio/tts/customization"
```

固定音色的模型和音色 ID 已写入数据库的 `voice_provider_profiles.provider_voice_id`，格式为 `bailian:qwen3-tts-flash:<voice>`。克隆音色成功后会写入个人音色，格式为 `bailian:qwen3-tts-vc-2026-01-22:<clone_voice_id>`。不要把 API Key 写入代码或提交到 Git；`.env` 已被 `.gitignore` 忽略。
