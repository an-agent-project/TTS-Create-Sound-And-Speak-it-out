# 启动说明

本文说明如何在本地启动前端、后端，以及初始化 MySQL 数据库。

## 1. 环境要求

- Node.js：用于启动 Vue + Vite 前端。
- Python 3.11+：用于 FastAPI 后端。
- MySQL 8.0：用于业务数据库。
- Windows PowerShell：以下命令按 PowerShell 编写。

## 2. 项目目录

进入项目根目录：

```powershell
cd C:\Users\lenovo\Desktop\TTS\TTS-Create-Sound-And-Speak-it-out
```

## 3. 前端启动

安装前端依赖：

```powershell
npm install
```

启动前端开发服务：

```powershell
npm run dev
```

默认访问地址：

```text
http://localhost:5173
```

前端的 Vite 配置已经把以下请求代理到后端：

```text
/api -> http://127.0.0.1:8000
/static -> http://127.0.0.1:8000
/audio -> http://127.0.0.1:8000
```

因此前端调用试听接口时，可以直接请求：

```text
/api/tts/preview
```

## 4. 后端虚拟环境

项目后端应使用项目根目录下的 `.venv`，不要直接使用 Anaconda base 环境。

创建虚拟环境：

```powershell
C:\Users\lenovo\anaconda3\python.exe -m venv .venv
```

安装后端依赖：

```powershell
.\.venv\Scripts\python.exe -m pip install -r backend\requirements.txt
```
配置数据库连接。当前本地 MySQL 可使用：

```powershell
$env:DATABASE_URL="mysql+pymysql://root:123456@127.0.0.1:3306/tts_podcast?charset=utf8mb4"
```

## 5. 后端启动

启动 FastAPI：

```powershell
.\.venv\Scripts\python.exe -m uvicorn app.main:app --app-dir backend --host 127.0.0.1 --port 8000
```

健康检查：

```text
http://127.0.0.1:8000/health
```

接口文档：

```text
http://127.0.0.1:8000/docs
```

当前已经实现的后端接口：

```text
# Auth
POST /api/auth/register    # 注册新账号，返回 JWT
POST /api/auth/login        # 登录，返回 JWT
GET  /api/auth/me           # 获取当前用户信息（需 Bearer token）
PUT  /api/auth/me           # 更新当前用户信息（手机/邮箱/头像）
POST /api/auth/change-password  # 修改密码（需 Bearer token）

# TTS
POST /api/tts/preview       # 音色试听（含数据库缓存）

# Voices
GET    /api/voices          # 音色列表（匿名仅见默认，登录后可见个人专属）
GET    /api/voices/{voice_id}  # 音色详情
POST   /api/voices          # 新增专属音色（需登录，自动归属当前用户）
PUT    /api/voices/{voice_id}  # 更新音色（仅 owner 可编辑）
DELETE /api/voices/{voice_id}  # 软删除音色（仅 owner 可删除）
```

## 用户认证机制

系统采用 JWT (JSON Web Token) 认证：

- 用户密码使用 bcrypt 哈希存储在 `users` 表，不存明文。
- 注册/登录成功后返回 `access_token`（有效期 24 小时）。
- 前端将 token 存储在 `localStorage` 的 `auth_token` 键中。
- 后续请求在 HTTP Header 中带 `Authorization: Bearer <token>`。
- 后端通过 `get_current_user` 依赖解析 token 并返回当前用户。

## 个人专属音色机制

`voices` 表通过 `owner_id` 字段区分系统默认音色与用户专属音色：

| `owner_id` | 含义 | 可见范围 | 编辑/删除 |
|-----------|------|---------|----------|
| `NULL` | 8 个系统默认音色 | 所有用户（含未登录） | ❌ 仅管理员 |
| 具体 user id | 用户个人专属音色 | 仅该用户自己 | ✅ 音色所有者 |

- 匿名用户 `GET /api/voices` 仅返回 8 个默认音色。
- 登录用户返回默认音色 + 自己创建的音色。
- 用户 A 看不到用户 B 的专属音色。
- `PUT` / `DELETE` 系统默认音色返回 403 Forbidden。

## 6. 数据库初始化

数据库采用 MySQL 8.0。初始化脚本位于：

```text
backend/db/init.sql
```

执行初始化：

```powershell
mysql -u root -p < backend\db\init.sql
```

执行后会创建数据库：

```text
tts_podcast
```

并创建以下表：

```text
users                       # 用户账号表
voices                      # 业务音色表
voice_provider_profiles     # TTS 供应商映射表
voice_preview_audios        # 试听音频缓存表
```

同时会写入当前音色库中的 Edge-TTS 音色种子数据，例如：

```text
晓晓 -> zh-CN-XiaoxiaoNeural
云希 -> zh-CN-YunxiNeural
晓伊 -> zh-CN-XiaoyiNeural
```

## 7. 当前数据库实现状态

目前已经完成：

- MySQL 初始化 SQL。
- 用户表 `users`（bcrypt 密码哈希 + JWT 认证）。
- 声音表 `voices`（含 `owner_id`，支持个人专属音色）。
- TTS 供应商映射表 `voice_provider_profiles`。
- 试听音频缓存表 `voice_preview_audios`。
- Edge-TTS 音色种子数据（8 个经过验证的音色）。
- 数据库连接配置。
- SQLAlchemy ORM 模型。
- 用户注册/登录/个人信息/密码修改接口。
- 声音数据库 CRUD 接口（含 owner 鉴权）。
- 试听接口读取数据库缓存。
- 未命中缓存时写入 `voice_preview_audios`。

目前尚未完成：

- 作品生成任务的数据库持久化。
- 长文本 Celery 异步任务。

也就是说，现在声音数据已经可以通过 `/api/voices` 做增删改查；试听音频也会通过 `voice_preview_audios` 做缓存读写。

## 8. 启动顺序

推荐按以下顺序启动：

1. 启动 MySQL。
2. 初始化数据库：

```powershell
mysql -u root -p < backend\db\init.sql
```

3. 启动后端：

```powershell
.\.venv\Scripts\python.exe -m uvicorn app.main:app --app-dir backend --host 127.0.0.1 --port 8000
```

4. 启动前端：

```powershell
npm run dev
```

5. 打开前端：

```text
http://localhost:5173
```

## Qwen 模型初始化

Qwen 模型文件不提交到 GitHub。首次运行或清理 `.hf-cache` 后，先执行以下命令下载/检查模型，并写入系统默认音色 `Qwen Vivian`：

```powershell
.\.venv\Scripts\python.exe backend\scripts\ensure_qwen_model.py
```

脚本会优先复用本地已有 snapshot；不存在时才通过 HuggingFace 下载 `Qwen/Qwen3-TTS-12Hz-0.6B-CustomVoice`。

## GPU 版 PyTorch 补充安装

`backend/requirements.txt` 只记录通用 Python 依赖，不固定 GPU 版 `torch` / `torchaudio`。GPU wheel 与本机显卡、NVIDIA 驱动和 CUDA 运行时相关，应在创建 `.venv` 并安装通用依赖后单独补装。

先确认显卡和驱动：

```powershell
nvidia-smi
```

常见显卡建议如下：

| 显卡类型 | 常见型号 | 建议安装 | 说明 |
| --- | --- | --- | --- |
| NVIDIA RTX 50 系 | RTX 5060 / 5070 / 5080 / 5090 | CUDA 12.8 wheel | 新架构优先使用较新的 PyTorch CUDA wheel，并保持显卡驱动为新版。 |
| NVIDIA RTX 40 系 | RTX 4050 / 4060 / 4070 / 4080 / 4090 | CUDA 12.6 或 12.8 wheel | 本项目当前 RTX 4050 可用 CUDA 12.6；6GB 显存跑 Qwen 0.6B 属于勉强可用，建议关闭其他占显存程序。 |
| NVIDIA RTX 30 系 | RTX 3050 / 3060 / 3070 / 3080 / 3090 | CUDA 12.6 wheel | 驱动较旧时可退回 CUDA 11.8 wheel。8GB 以上显存体验更稳。 |
| NVIDIA RTX 20 系 / GTX 16 系 | RTX 2060 / 2070 / 2080，GTX 1660 | CUDA 11.8 或 12.6 wheel | 优先看 `nvidia-smi` 驱动版本；老驱动优先 CUDA 11.8。显存不足时不建议跑 Qwen 本地生成。 |
| NVIDIA GTX 10 系 | GTX 1060 / 1070 / 1080 | CUDA 11.8 wheel 或 CPU | 可以尝试 CUDA 11.8，但显存和算力通常不适合 Qwen TTS，本项目建议仅做 Edge-TTS 或轻量测试。 |
| AMD / Intel / 无独显 | Radeon / Arc / 核显 | CPU wheel | Windows 下本项目暂不按 ROCm/DirectML 路线配置，Qwen 本地生成会很慢或不可用。 |

RTX 30/40 系常用安装命令：

```powershell
.\.venv\Scripts\python.exe -m pip uninstall -y torch torchvision torchaudio
.\.venv\Scripts\python.exe -m pip install torch torchaudio --index-url https://download.pytorch.org/whl/cu126
```

RTX 50 系或较新驱动可尝试 CUDA 12.8：

```powershell
.\.venv\Scripts\python.exe -m pip uninstall -y torch torchvision torchaudio
.\.venv\Scripts\python.exe -m pip install torch torchaudio --index-url https://download.pytorch.org/whl/cu128
```

老驱动或 RTX 20 / GTX 16 / GTX 10 系可尝试 CUDA 11.8：

```powershell
.\.venv\Scripts\python.exe -m pip uninstall -y torch torchvision torchaudio
.\.venv\Scripts\python.exe -m pip install torch torchaudio --index-url https://download.pytorch.org/whl/cu118
```

CPU 兜底安装：

```powershell
.\.venv\Scripts\python.exe -m pip uninstall -y torch torchvision torchaudio
.\.venv\Scripts\python.exe -m pip install torch torchaudio
```

安装后验证：

```powershell
.\.venv\Scripts\python.exe -c "import torch; print(torch.__version__); print(torch.cuda.is_available()); print(torch.version.cuda); print(torch.cuda.get_device_name(0) if torch.cuda.is_available() else 'none')"
```

正常使用 GPU 时，`torch.cuda.is_available()` 应输出 `True`，并能看到显卡名称。若输出 `False`，通常是装成了 CPU 版 PyTorch、NVIDIA 驱动过旧，或当前命令使用的不是项目 `.venv`。

官方安装命令以 PyTorch 官网为准：`https://pytorch.org/get-started/locally/`。
