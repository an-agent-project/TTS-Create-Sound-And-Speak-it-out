# 有声读物智能生成系统

## 项目简介

有声读物智能生成系统是一个面向内容创作者的 Web 端 AI 配音工具，目标是帮助自媒体作者、知识付费讲师、播客主播和教育内容生产者快速将文本内容转换为可发布的有声作品。

系统以“选择场景 -> 粘贴/上传文本 -> 选择音色 -> 一键生成 -> 预览导出”为核心流程，降低传统配音对录音设备、配音经验、后期剪辑和制作成本的依赖。

## 项目价值

- 零门槛：用户无需专业录音设备和配音经验，输入文本即可生成音频。
- 高效率：通过文本预处理、TTS 合成和音频拼接自动化，缩短内容制作周期。
- 个性化：提供预设音色库、场景模板、语速/音调/情感参数调节。
- 低成本：优先使用 Edge-TTS 等轻量方案，降低项目部署和运行成本。

## 目标用户

- 自媒体内容创作者
- 知识付费讲师
- 播客主播
- 教育内容生产者
- 需要批量生成音频内容的小团队

## 核心功能

### 创作工作台

- 场景选择：播客模式、知识讲解、故事讲述、情感朗读。
- 文本输入：支持文本粘贴和 TXT/DOC 文件上传。
- 音色选择：按性别、风格、场景筛选预设音色，支持试听和收藏。
- 参数调节：支持语速、音调、情感标签等轻量调节。
- 背景音乐：支持按氛围添加 BGM，并进行音量、淡入淡出处理。
- 合成预览：支持片段试听、全文生成、进度展示。
- 导出下载：支持 MP3、WAV 导出，可扩展封面生成。

### AI 服务能力

- 文本预处理：文本清洗、智能分段、停顿标注、敏感词检测。
- 语音合成：基于 Edge-TTS，备选阿里云基础 TTS。
- 情感处理：基于规则的情感识别和情感标签映射。
- 音频合成：多段音频拼接、背景音乐叠加、音量均衡、格式转换。

### 基础支撑

- 用户认证与个人中心
- 作品管理与历史记录
- 文件存储与音频资源管理
- 异步任务队列
- 日志与基础监控

## 技术栈概览

- 前端：Vue 3.5、TypeScript 5.7、Vite 8、Element Plus 3、TailwindCSS 4、Pinia 3、Howler.js。
- 后端：FastAPI 0.115+、Pydantic、SQLAlchemy、MySQL 8.0、Redis、JWT、Docker。
- AI 与音频：Edge-TTS、阿里云基础 TTS（备选）、pydub、FFmpeg、Celery + Redis。

详细技术栈见 [技术选型确认书](./技术选型确认书.md)。

## 本地运行

### 1. 启动后端

```bash
cd backend
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
uvicorn app.main:app --reload --port 8000
```

后端默认提供：

- API 地址：http://127.0.0.1:8000/api
- 接口文档：http://127.0.0.1:8000/docs
- 音频文件：http://127.0.0.1:8000/media

### 2. 启动前端

```bash
npm install
npm run dev
```

然后打开：http://localhost:5173

前端已配置 Vite 代理，`/api` 和 `/media` 会转发到本地 FastAPI 后端。

## 后端已实现接口

- `GET /api/health`：后端健康检查
- `GET /api/scenes`：获取场景模板
- `GET /api/voices`：获取中文 Edge-TTS 音色库
- `POST /api/text/preprocess`：文本清洗、智能分段、停顿估计、敏感词检测
- `POST /api/tts/generate`：调用 Edge-TTS 生成 MP3，并自动保存作品
- `GET /api/works`：获取作品列表
- `GET /api/works/{work_id}`：获取作品详情
- `DELETE /api/works/{work_id}`：删除作品和对应音频文件

当前版本使用 `backend/data/works.json` 保存作品元数据，使用 `backend/media/` 保存生成的音频文件。BGM 参数已在接口中保留，后续可接入 FFmpeg/pydub 完成背景音乐混音。
