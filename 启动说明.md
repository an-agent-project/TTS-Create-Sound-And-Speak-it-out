# 启动说明

本文说明如何在本地启动前端、后端，以及初始化 MySQL 数据库。

## 1. 环境要求

- Node.js：用于启动 Vue + Vite 前端。
- Python 3.11+：用于 FastAPI 后端。
- MySQL 8.0：用于业务数据库。
- Windows PowerShell：以下命令按 PowerShell 编写。

## 2. 项目目录

进入项目根目录：

```powershell
cd C:\Users\lenovo\Desktop\TTS\TTS-Create-Sound-And-Speak-it-out
```

## 3. 前端启动

安装前端依赖：

```powershell
npm install
```

启动前端开发服务：

```powershell
npm run dev
```

默认访问地址：

```text
http://localhost:5173
```

前端的 Vite 配置已经把以下请求代理到后端：

```text
/api -> http://127.0.0.1:8000
/static -> http://127.0.0.1:8000
```

因此前端调用试听接口时，可以直接请求：

```text
/api/tts/preview
```

## 4. 后端虚拟环境

项目后端应使用项目根目录下的 `.venv`，不要直接使用 Anaconda base 环境。

创建虚拟环境：

```powershell
C:\Users\lenovo\anaconda3\python.exe -m venv .venv
```

安装后端依赖：

```powershell
.\.venv\Scripts\python.exe -m pip install -r backend\requirements.txt
```
配置数据库连接。当前本地 MySQL 可使用：

```powershell
$env:DATABASE_URL="mysql+pymysql://root:123456@127.0.0.1:3306/tts_podcast?charset=utf8mb4"
```

## 5. 后端启动

启动 FastAPI：

```powershell
.\.venv\Scripts\python.exe -m uvicorn app.main:app --app-dir backend --host 127.0.0.1 --port 8000
```

健康检查：

```text
http://127.0.0.1:8000/health
```

接口文档：

```text
http://127.0.0.1:8000/docs
```

当前已经实现的后端接口：

```text
POST /api/tts/preview
GET /api/voices
GET /api/voices/{voice_id}
POST /api/voices
PUT /api/voices/{voice_id}
DELETE /api/voices/{voice_id}
```

该接口接收试听文本和 Edge-TTS 音色 ID，生成试听音频，并返回静态音频地址。
试听接口已经接入 MySQL：

- 先通过 `voice_provider_profiles.provider_voice_id` 查找音色供应商配置。
- 再通过 `voice_preview_audios` 查询是否已有相同试听文本的缓存。
- 命中缓存时直接返回数据库中的 `audio_url`。
- 未命中缓存时调用 Edge-TTS 生成 MP3，并把音频路径、URL、时长、文件大小写入 `voice_preview_audios`。

## 6. 数据库初始化

数据库采用 MySQL 8.0。初始化脚本位于：

```text
backend/db/init.sql
```

执行初始化：

```powershell
mysql -u root -p < backend\db\init.sql
```

执行后会创建数据库：

```text
tts_podcast
```

并创建以下表：

```text
voices
voice_provider_profiles
voice_preview_audios
```

同时会写入当前音色库中的 Edge-TTS 音色种子数据，例如：

```text
晓晓 -> zh-CN-XiaoxiaoNeural
云希 -> zh-CN-YunxiNeural
晓伊 -> zh-CN-XiaoyiNeural
```

## 7. 当前数据库实现状态

目前已经完成：

- MySQL 初始化 SQL。
- 声音表 `voices`。
- TTS 供应商映射表 `voice_provider_profiles`。
- 试听音频缓存表 `voice_preview_audios`。
- Edge-TTS 音色种子数据。
- 数据库连接配置。
- SQLAlchemy ORM 模型。
- 声音数据库 CRUD 接口。
- 试听接口读取数据库缓存。
- 未命中缓存时写入 `voice_preview_audios`。

目前尚未完成：

- 作品生成任务的数据库持久化。
- 长文本 Celery 异步任务。

也就是说，现在声音数据已经可以通过 `/api/voices` 做增删改查；试听音频也会通过 `voice_preview_audios` 做缓存读写。

## 8. 启动顺序

推荐按以下顺序启动：

1. 启动 MySQL。
2. 初始化数据库：

```powershell
mysql -u root -p < backend\db\init.sql
```

3. 启动后端：

```powershell
.\.venv\Scripts\python.exe -m uvicorn app.main:app --app-dir backend --host 127.0.0.1 --port 8000
```

4. 启动前端：

```powershell
npm run dev
```

5. 打开前端：

```text
http://localhost:5173
```

## 9. 验证命令

运行后端测试：

```powershell
.\.venv\Scripts\python.exe -m pytest backend\tests -v
```

运行前端构建：

```powershell
npm run build
```

手动测试试听接口：

```powershell
Invoke-WebRequest `
  -Uri http://127.0.0.1:8000/api/tts/preview `
  -Method POST `
  -ContentType "application/json" `
  -Body '{"text":"大家好，欢迎收听本期节目。","voiceId":"zh-CN-XiaoxiaoNeural"}'
```

成功时会返回类似：

```json
{
  "audioUrl": "/static/previews/preview-xxxx.mp3",
  "duration": 5
}
```
