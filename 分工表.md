# 有声读物智能生成系统分工表

## 分工原则

本项目采用“传统前端 + 传统后端 + AI工程师”的三类角色分工。前端负责用户可见的创作体验，后端负责稳定的业务服务和数据支撑，AI 工程师负责文本处理、TTS、音频生成等智能能力。

## 角色职责总览

| 角色 | 负责模块 | 具体职责 | 主要交付物 |
|-----|---------|---------|-----------|
| 传统前端工程师 | 用户端创作平台 | 首页、创作工作台、音色库、我的作品、个人中心页面开发 | Web 前端页面、组件库、前端路由、状态管理 |
| 传统后端工程师 | API 服务与基础支撑 | 用户认证、作品管理、文件存储、数据库、任务队列、接口文档、部署 | 后端 API、数据库 Schema、任务系统、部署配置 |
| AI 工程师 | AI 服务与音频处理 | 文本预处理、TTS 接入、情感标签映射、音频拼接、BGM 叠加、导出优化 | AI 服务模块、TTS 测试报告、音频处理流水线 |

## 传统前端工程师分工

| 模块 | 具体任务 | 协作对象 |
|-----|---------|---------|
| 首页与入口 | 项目介绍、场景入口、案例展示、快速开始入口 | 后端提供案例数据接口 |
| 创作工作台 | 场景选择、文本输入、音色选择、参数调节、BGM 选择、生成按钮、进度展示 | 后端提供任务接口，AI 工程师确认参数含义 |
| 音色库 | 音色列表、分类筛选、在线试听、收藏常用音色 | 后端提供音色数据和收藏接口 |
| 我的作品 | 作品列表、播放、下载、重新编辑、删除 | 后端提供作品管理接口 |
| 音频播放体验 | 使用 Howler.js 实现试听、播放、暂停、进度控制 | AI 工程师提供试听音频或生成接口 |
| 前端工程化 | Vue 3 项目结构、路由、Pinia 状态管理、接口封装、错误提示 | 后端统一 API 规范 |

## 传统后端工程师分工

| 模块 | 具体任务 | 协作对象 |
|-----|---------|---------|
| 用户认证 | 注册登录、JWT 鉴权、用户信息、权限校验 | 前端对接登录态 |
| 作品管理 | 作品创建、查询、更新、删除、状态流转、下载地址管理 | 前端展示作品状态 |
| 音色与场景数据 | 音色库、场景模板、默认参数、收藏关系维护 | 前端展示筛选和推荐结果 |
| 文件存储 | 文本文件上传、生成音频存储、本地/OSS 扩展 | AI 工程师读取和写入音频文件 |
| 任务队列 | 创建合成任务、查询任务进度、失败重试、异步状态更新 | AI 工程师执行生成任务，前端轮询或 WebSocket 展示进度 |
| 数据库设计 | 用户表、音色表、场景模板表、作品表、收藏表 | 全员确认字段满足业务流程 |
| 部署与接口文档 | Docker 配置、环境变量、API 文档、日志配置 | 前端和 AI 工程师按接口联调 |

## AI 工程师分工

| 模块 | 具体任务 | 协作对象 |
|-----|---------|---------|
| 文本预处理 | 文本清洗、智能分段、长句拆分、停顿标注、敏感词检测 | 前端展示预处理结果，后端存储结构化文本 |
| TTS 服务接入 | Edge-TTS 接入、阿里云基础 TTS 备选方案、音色参数映射 | 后端封装任务调用接口 |
| 情感处理 | 平静、开心、悲伤、激动等情感标签到语速/音调参数的映射 | 前端提供情感选择控件 |
| 音频合成 | 多段语音拼接、停顿插入、音量均衡、格式转换 | 后端管理任务和文件路径 |
| 背景音乐处理 | BGM 叠加、音量控制、淡入淡出、氛围分类 | 前端提供 BGM 选择，后端提供素材管理 |
| 质量优化 | 生成稳定性测试、长文本分段策略、失败重试策略、效果评估 | 后端实现任务重试，前端展示异常信息 |

## 阶段性协作安排

| 阶段 | 时间 | 传统前端 | 传统后端 | AI 工程师 |
|-----|------|----------|----------|-----------|
| 第一阶段：需求与环境 | 第 1 周 | 页面原型、组件规范、前端脚手架 | API 规范、数据库设计、后端脚手架 | TTS 调研、音色清单、文本处理方案 |
| 第二阶段：核心能力 | 第 2 周 | 音色库、场景选择、基础创作页面 | 音色/场景/作品接口、任务接口 | TTS 封装、文本预处理、基础音频生成 |
| 第三阶段：业务闭环 | 第 3 周 | 工作台完整交互、作品管理、音频播放 | 异步任务、文件存储、下载接口 | BGM 叠加、音频拼接、格式导出 |
| 第四阶段：联调交付 | 第 4 周 | UI 细节、响应式适配、异常提示 | 性能优化、部署、接口文档完善 | 生成质量优化、稳定性测试、演示素材准备 |

## 关键接口协作点

| 协作点 | 前端输入 | 后端处理 | AI 输出 |
|-------|----------|----------|---------|
| 创建合成任务 | 文本、场景、音色、语速、音调、情感、BGM | 创建任务并入队 | 返回任务处理结果和音频文件 |
| 查询任务进度 | task_id | 查询任务状态 | processing/completed/failed 状态 |
| 音色试听 | voice_id | 返回试听地址或触发试听生成 | 生成短音频试听 |
| 作品下载 | work_id | 校验权限并返回文件 | MP3/WAV 文件 |

## 责任边界

- 前端不直接调用第三方 TTS 服务，统一通过后端 API 调用。
- 后端不实现复杂音频算法，只负责服务编排、数据存储、任务调度和权限控制。
- AI 工程师负责生成效果和音频处理逻辑，并以服务函数或任务处理器形式供后端调用。
